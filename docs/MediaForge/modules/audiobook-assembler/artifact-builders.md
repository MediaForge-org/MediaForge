## Gemeinsame VertrΟΛge

**Input-Signatur** (Idempotenz-SchlΟΦssel aller Builder): BLAKE3 ΟΦber die kanonische Serialisierung von (geordnete Track-Content-Hashes der Sequenz, aktives Chapter Set als `(seq, start_ms, end_ms, title)`-Liste, Builder-Kennung + normalisierte Parameter, Builder-Formatversion). **Determinismus:** Alle Text-Writer (`CueWriter`, `FfmetadataWriter`, ABS-`metadata.json`) sind byte-deterministisch: feste Zeilenenden (`\r\n` fΟΦr CUE βÄî der verbreitetste Player-KompatibilitΟΛtsnenner; `\n` sonst), UTF-8 ohne BOM (Ausnahme CUE: BOM als Setting `assembler.builders.cue_bom`, Default false, fΟΦr ΟΛltere Windows-Player), keine Zeitstempel im Inhalt (Erzeugungszeit gehΟΕrt ins Artefakt-Metadatum, nicht in die Datei βÄî sonst bricht die Idempotenz).

**Namensschema** (unter `/artifacts/audiobooks/{edition_ulid}/`): `{basename}.cue`, `{basename}.ffmeta`, `{basename}.m4b`, `abs-export/` βÄî `basename` = slugifizierter Werktitel + Kurz-ULID (kollisionsfest, dateisystem-sicher, NFC).

Ein Kapitel, dessen Grenze **nicht** auf einer Dateigrenze liegt, beginnt als TRACK mitten in der laufenden FILE (Modulkapitel); Kapitel, die mehrere Dateien ΟΦberspannen, erzeugen **keine** zusΟΛtzlichen TRACKs βÄî CUE modelliert Startmarken, die Fortsetzung ΟΦber Dateigrenzen ist implizit. **Titel-Escaping:** doppelte AnfΟΦhrungszeichen βÜ£ einfache; Nicht-ASCII bleibt (UTF-8-CUE ist 2026er-RealitΟΛt; das `cue_ascii_fallback`-Setting transliteriert fΟΦr Legacy-Hardware). `PERFORMER` = Autor, `REM MediaForge_NARRATOR` = Sprecher (kein Standard-Feld vorhanden). ### Dialekt (b): eine CUE pro Quelldatei

Je Datei mit βâΞ 1 Kapitelstart eine `{trackbasename}.cue` mit FILE-lokalen Marken; Kapitel ohne Marke in einer Datei (Fortsetzungs-Dateien) erhalten **keine** CUE (leere CUEs verwirren Player). Anwendungsfall: Player, die Sidecar-CUEs nur dateiweise lesen. Beide Dialekte kΟΕnnen koexistieren (getrennte Artefakte, getrennte Signaturen).

### Kommandomuster

```bash
# Schritt concat (COPY-Pfad):
ffmpeg -f concat -safe 0 -i tracks.txt -c copy -movflags +faststart part.m4a
# Schritt transcode (je Chunk c von 1..K):
ffmpeg -i chunk_c.wavpipe -c:a libfdk_aac -profile:a aac_he -b:a 64k chunk_c.m4a
# Schritt mux:
ffmpeg -i merged.m4a -i meta.ffmeta -i cover.jpg \
       -map 0:a -map 2:v -map_metadata 1 -c:a copy -c:v mjpeg \
       -disposition:v attached_pic -movflags +faststart out.partial.m4b
```

`tracks.txt`-EintrΟΛge nutzen escaped absolute Pfade **innerhalb des media-tools-Containers** (der Builder lΟΛuft dort; PHP orchestriert nur βÄî Stack-Regel: PHP fΟΦhrt kein ffmpeg aus). `-movflags +faststart` ist Pflicht (Streaming-taugliches moov-Atom vorn); Cover aus dem Katalog (grΟΕΟütes Cover-Asset, JPEG-Rekompression nur bei > 2 MB).

### Chunking und Checkpoints

TRANSCODE teilt die Sequenz in Chunks entlang Trackgrenzen mit ZielgrΟΕΟüe ~2 h Audio (`assembler.builders.m4b_chunk_hours`, Default 2). ResumableJob-Schritte: `plan` (Chunk-Zuschnitt, persistiert), `chunk-{n}` (je Chunk ein Schritt βÄî Wiederaufnahme beginnt beim ersten fehlenden Chunk-Ergebnis; Chunk-Artefakte leben unter dem Job-Arbeitsverzeichnis mit Content-Hash-PrΟΦfung), `merge` (concat der Chunk-M4As, copy), `mux`, `verify`. Chunk-Grenzen an Trackgrenzen halten die Wiederaufnahme sample-exakt (kein Anschneiden von Frames zwischen LΟΛufen).

### Verifikation (`verify`)

ffprobe auf dem `.partial`: Laufzeit |Έî| βâΛ 1 s gegen Timeline; Kapitelzahl = Set-Kapitelzahl; erste/letzte Kapitelgrenze exakt; Tag-Kernfelder vorhanden; Datei > 90 % der Prognose (Transcode-Abbruch-Detektor). Ein Verifikationsfehler lΟΛsst das `.partial` zur Diagnose liegen (AufrΟΛumung durch den Artefakt-GC nach 7 Tagen) und endet als Fachfehler mit Review `artifact_verification_failed`.

## Staleness und Rebuild-Politik

Artefakte tragen ihre Signatur; die Assembly kennt die Signatur des aktuellen Zustands. Divergenz β΅£ UI-Staleness-Badge (Modulkapitel) und Health-Metrik `audiobook_artifacts_stale`. **Kein Auto-Rebuild** (Modulkapitel-BegrΟΦndung: Stunden-Kosten); die dokumentierte Automatisierung ist eine Rule-Engine-Regel (Beispiel im [Definitions-Katalog](../workflow-engine/definitions-catalog.md)): `on ChapterSetActivated if artifact_stale(m4b) and setting(auto_rebuild) then enqueue BuildM4bJob` βÄî Opt-in des Betreibers, auditiert wie jede Regel-AusfΟΦhrung.
